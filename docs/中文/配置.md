# 本文将讲述如何配置`command.json`

## 示例
```json
{
  "status": "-",
  "scan": "- <mode> [timeout:10] @targets(^[a-z]+$):3",
  "upload": "- @files(\\.png$)"
}
```

## 参数概念
`"script": "这里是参数格式,参数格式里面填充参数,参数与参数之间用空格分隔"`
| 符号 | 示例 | 定义效果 |
| :---: | :--- | :--- |
| **`-`** | `-` | 表示这个命令自身 |
| **`<必填参数>`** | `<file>` | 强制要求用户输入,缺失则报错 |
| **`[可填参数]`** | `[debug]` | 用户可输可不输,缺失为 `None` |
| **`[可填参数:默认值]`** | `[port:8080]` | 用户未输入时,自动填充默认值 |
| **`@数组参数`** | `@src` | 自动收集剩余所有参数为列表 |
| **`@数组参数(正则表达式)`** | `@log(\.txt$)` | 列表项必须匹配正则,否则置为 `None` |
| **`@数组参数:长度`** | `@vec:3` | 强制列表长度为 3,不足补 `None` |

**注意**:每个参数格式前面必须加上`-`或者`- `,如果后面有其他参数使用`- `,没有就用`-`  
如果参数格式只写了`-`则会直接调用`enter`函数  
数组参数的正则表达式还可以配合长度使用:`- @args(\d+):5`,表示取后5个参数且符合正则表达式的所有参数

## 命令ID
`"这里是命令的ID,用来明确该命令在哪里调用": "format"`  
此处需要注意的是:ID必须在你的`setting.json`中的`command_config`所指的文件中写入  
且ID必须用`"."`号分隔,比如`file.add`表示在`command_dir/file/add.py`  
那么`file`就表示在`command_dir/file.py`

# 本文将讲述如何配置`command.json`

## command.json 配置说明

`command.json` 是命令配置文件，用于定义所有可用命令的参数格式和类型。该文件位于 `PATH` 目录下，具体路径由 `setting.json` 中的 `command_config` 字段指定。

### 文件格式

```json
{
    "命令ID": "命令配置字符串",
    "命令ID.子命令": "命令配置字符串"
}
```

命令 ID 使用点号分隔表示层级，但在命令行中使用空格分隔。

### 命令配置字符串语法

支持两种参数类型：
- 必填参数：`<参数名(正则表达式):长度>类型`
- 可选参数：`[参数名(正则表达式):长度=默认值]类型`

#### 参数说明
- **参数名**：字母、数字、下划线组成，必须以字母或下划线开头
- **正则表达式**（可选）：用括号包裹，在 JSON 中需要双重转义
- **长度**（可选）：`:数字` 表示固定长度数组，`:0` 表示接收剩余所有参数
- **默认值**（可选）：仅可选参数可用，格式 `=值`
- **类型**（可选）：`string`（默认）、`int`、`float`、`bool`、`json`

### 示例

#### 基础示例
```json
{
    "user": "<action>",
    "user.add": "<name> [age]int",
    "user.delete": "<user_id>int [force]bool",
    "user.list": "[page=1]int [size=20]int"
}
```

#### 正则验证（注意双重转义）
```json
{
    "user.add": "<username(^[a-zA-Z][a-zA-Z0-9_]{3,15}$)> [email(^[\\w\\.-]+@[\\w\\.-]+\\.[\\w]+$)]",
    "user.date": "<date(^\\d{4}-\\d{2}-\\d{2}$)>",
    "user.phone": "<phone(^1[3-9]\\d{9}$)>"
}
```

#### 数组参数
```json
{
    "user.batch": "<ids:3>int",
    "user.search": "<keyword> [tags:0]",
    "user.multi": "<values:0>float"
}
```

#### 复杂类型
```json
{
    "data.import": "<config>json",
    "user.add": "<name> [roles:0]json [age=0]int"
}
```

# 本文将讲述config函数API的所有tools的用法

## Tran
`Tran` 是一个国际化翻译类，用于根据当前语言设置获取对应的翻译文本。

### 初始化
```python
tran = Tran(translate_map: dict, lang: str)
```
- **translate_map**：多语言字典，结构为 `{ "语言代码": { "key": "翻译文本" } }`
- **lang**：当前使用的语言代码（如 `"zh-cn"`、`"en-us"`）

### 方法

#### `run(key: str, content: str = "<?>") -> str`
获取指定键的翻译文本，并将 `content` 中的 `"<?>"` 替换为翻译内容。

- **key**：翻译键名
- **content**：包含占位符 `"<?>"` 的字符串，默认为 `"<?>"`

### 示例
```python
TRAN = {
    "zh-cn": {
        "hello": "你好",
        "welcome": "欢迎, <?>"
    },
    "en-us": {
        "hello": "Hello",
        "welcome": "Welcome, <?>"
    }
}

tran = Tran(TRAN, "zh-cn")
print(tran.run("hello"))            # 输出: 你好
print(tran.run("welcome", "<?> 用户"))  # 输出: 欢迎, 用户
```

## config_parser
解析命令配置字符串，将参数定义转换为结构化数据。

### 函数签名
```python
def config_parser(config: str) -> list[dict]
```

### 参数
- **config**：配置字符串，例如 `<name> [age(\d+)]int`

### 返回
返回一个字典列表，每个字典代表一个参数定义。

**字典字段说明：**
- **class**：参数类型，`1` 表示必填参数 `<...>`，`2` 表示可选参数 `[...]`
- **name**：参数名
- **regex**：正则表达式校验规则（可选）
- **length**：数组长度（可选，`0` 表示捕获剩余所有参数）
- **default**：默认值（仅 `class=2` 时存在）
- **type**：参数类型（`string`、`int`、`float`、`bool`、`json`）

### 示例
```python
config = r"<name> [age(\d+)]int [tags:0]json"
result = config_parser(config)
# 返回:
# [
#   {"class": 1, "name": "name", "regex": None, "length": None, "type": "string"},
#   {"class": 2, "name": "age", "regex": None, "length": None, "default": None, "type": "int"},
#   {"class": 2, "name": "tags", "regex": None, "length": 0, "default": None, "type": "json"}
# ]
```

## to_type
将字符串转换为指定类型，支持类型转换和异常处理。

### 函数签名
```python
def to_type(text: str | None, type_: str | None) -> any | None
```

### 参数
- **text**：要转换的字符串
- **type_**：目标类型（`string`、`int`、`float`、`bool`、`json`）

### 返回
转换后的值，如果转换失败或参数无效则返回 `None`。

### 示例
```python
print(to_type("123", "int"))        # 输出: 123
print(to_type("3.14", "float"))     # 输出: 3.14
print(to_type("true", "bool"))      # 输出: True
print(to_type('{"a":1}', "json"))   # 输出: {'a': 1}
print(to_type("abc", "int"))        # 输出: None
```

## run_func
执行命令函数，根据配置解析命令行参数并调用目标函数。

### 函数签名
```python
def run_func(enter, config: str, arg_start_index: int) -> None
```

### 参数
- **enter**：要执行的函数（通常为命令模块的 `enter` 函数）
- **config**：命令配置字符串（格式与 `config_parser` 相同）
- **arg_start_index**：参数起始索引（从 `sys.argv` 中的位置开始）

### 功能
1. 解析 `config` 生成参数定义
2. 根据参数定义从 `args` 中提取值
3. 对值进行类型转换和正则校验
4. 使用 `**kwargs` 调用 `enter` 函数

### 示例
假设有命令配置 `"<name> [age:int]"` 和 `args = ["John", "25"]`，执行后相当于调用：
```python
enter(name="John", age=25)
```

## AdminCommands
管理员命令类，提供帮助信息查看和命令创建功能。

### 初始化
```python
admin = AdminCommands(debug: bool = False)
```

### 方法

#### `help(id: str | None) -> None`
显示命令帮助信息。

- **id**：命令 ID，若为 `None` 则列出所有命令

#### `create(id: str | None, config: str | None) -> None`
创建新命令文件或更新命令配置。

- **id**：命令 ID（如 `"user.add"`）
- **config**：命令配置字符串（如 `"<name> [age:int]"`）
- 当 `id` 和 `config` 均为 `None` 时，根据现有配置生成命令文件

### 示例
```python
admin = AdminCommands(debug=True)

# 显示所有命令帮助
admin.help(None)

# 显示特定命令帮助
admin.help("user.add")

# 创建或更新命令
admin.create("user.add", "<name> [age:int]")
```

## run_admin_func
运行管理员命令的入口函数。

### 函数签名
```python
def run_admin_func(admin_args: list[str]) -> None
```

### 参数
- **admin_args**：管理员命令参数列表（不包含 `--admin`）

### 功能
1. 初始化 `AdminCommands` 实例
2. 匹配管理员命令（如 `help`、`create`）
3. 调用对应的管理员命令函数
4. 如果命令不存在则报错并退出

### 示例
假设命令行输入：
```bash
python main.py --admin help user.add
```
此时 `admin_args = ["help", "user.add"]`，`run_admin_func` 会调用 `admin.help("user.add")`。
